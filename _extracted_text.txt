1.概述


1.1 机器狗设计意义


（1）科学研究与技术创新


仿生学研究：机器狗的设计借鉴了动物的运动方式和行为模式，通过模拟狗的腿部结构和运动控制算法，能够为仿生学研究提供实际的实验平台。研究其运动学和动力学模型，有助于深入理解生物体的运动机制。


多学科融合：机器狗的设计涉及电子工程、机械工程、计算机科学、控制理论等多个学科领域。通过跨学科的综合应用，推动相关技术的协同发展，促进学科交叉创新。


（2）教育与培训


实践教学平台：机器狗实训课程为我们提供了一个综合性的实践平台，涵盖了硬件设计、软件编程、通信技术、机械结构等多个方面的知识。通过实际操作，让我们能够将理论知识与实践相结合，加深对专业知识的理解和掌握。


创新能力培养：在设计和实现机器狗的过程中，我们需要解决各种实际问题，如硬件故障排除、软件调试优化、动作控制算法设计等。这有助于培养我们的创新思维和问题解决能力。


团队协作能力：机器狗的设计和调试通常需要多人协作完成，我们在项目中学会分工合作、沟通交流，提升团队协作能力。这种协作能力对于未来的职业发展具有重要意义。


1.2 课程设计的组成部分

本设计基于STM32微控制器实现机器狗的基础功能，主要包含以下模块：

主控模块：

（1）STM32F4系列单片机（如STM32F407），负责整体逻辑控制、外设调度。

输入模块：

（1）按键输入（GPIO轮询/外部中断）

（2）串口指令（PC端控制命令）

执行模块：

（1）舵机控制（PCA9685驱动，通过I2C通信）

（2）蜂鸣器驱动（TIM5 PWM输出）

（3）LED呼吸灯（TIM2 PWM动态调光）

通信模块：

（1）UART串口通信（与PC端交互指令）

（2）I2C通信（控制PCA9685舵机驱动板）

辅助功能模块：

（1）定时器中断（TIM2用于LED动态效果）

电源模块：

（1）外部电源供电（使用锂电池+稳压电路）


1.2.1 设计原理

主控逻辑：采用事件驱动架构，解析串口指令，驱动舵机和LED实现动作反馈。

PWM控制：舵机：通过PCA9685生成周期20ms的PWM信号，调节占空比（0.5ms~2.5ms）对应0°~180°角度。呼吸灯：TIM2动态调整CCR值，改变LED亮度，模拟呼吸效果。

通信协议：UART：ASCII指令格式（如LED RED ON），通过HAL_UARTEx_ReceiveToIdle接收数据。I2C：通过HAL库函数与PCA9685通信，设置舵机角度。


1.2.2 各部分电路的作用

GPIO输入电路：

（1）按键连接至GPIOF端口（PF5-PF9），内部上拉，按下时拉低电平，触发中断或轮询检测。

PWM输出电路：

（1）舵机驱动：PCA9685通过I2C接收指令，输出多路PWM信号至舵机，控制机械腿运动。

（2）LED调光：TIM2通道1-3输出PWM至RGB LED，实现颜色混合与亮度渐变。

I2C通信电路：

（1）SCL（PB10）、SDA（PB11）连接PCA9685，配置舵机角度，支持多设备寻址。

UART通信电路：

（1）USART1（PA9-TX, PA10-RX）通过USB转TTL与PC通信，波特率115200bps。

电源电路：

（2）采用7.3V锂电池，为MCU、舵机、外设供电。


1.2.3 设计方案的的分析

优势：

（1）模块化设计：代码按功能模块划分（如UART、PWM、I2C），便于扩展和维护。

（2）实时响应：外部中断实现按键快速检测，避免轮询延迟。

（3）灵活控制：PWM动态调节支持舵机精准角度、蜂鸣器多音调播放。

（4）调试友好：集成日志系统，实时输出运行状态，便于故障排查。

不足与改进：

（1）阻塞式延时：呼吸灯代码使用HAL_Delay导致主循环卡顿（应改用定时器中断非阻塞更新PWM）。

（2）通信协议简单：ASCII指令易受干扰（可升级为二进制协议+CRC校验，如0xAA [CMD] [DATA] [CRC]）。

（3）舵机校准不足：角度范围未做限制，可能损坏机械结构（需添加角度边界检查，如if(angle>180) angle=180）。

2.2 软件设计


2.2.1 运动学逆解程序设计

模块说明：

本模块实现了对机器狗腿部舵机角度的逆解算，根据脚尖的目标位置（x, y 坐标）反推出两组舵机的目标角度，从而使腿部精确移动到该位置，通过余弦定理、三角函数等几何计算方法实现。

函数逻辑设计：

InverseResolve 函数根据输入的足尖坐标 (x, y) 计算出大腿和小腿的舵机角度。首先，通过余弦定理计算小腿的弯曲角度，并根据该角度计算小腿舵机的角度。如果小腿的运动范围超出限制，则返回 -1 表示不可达。接着，使用几何关系计算大腿的舵机角度，特殊情况下对 x 为零的情况进行处理。最后，若所有计算成功且目标位置可达，返回 0，否则返回 -1。


2.2.2 动作控制模块程序设计

模块说明：

本模块实现了机器狗的运动控制逻辑，通过解析一系列预定义的动作命令序列，控制机器狗的腿部舵机角度，进而使机器狗完成指定的动作，如前进、后退、转向、站立、蹲下等。通过与舵机的角度控制接口（PCA9685）进行交互，结合运动学逆解计算，确保机器狗动作的精确执行。

函数逻辑设计：

（1）SetAngelByID： 该函数根据传入的腿部 ID 和舵机角度（A1, A2）来设置相应舵机的角度，同时附加了一个舵机的角度偏置（Px[]）。通过不同的舵机 ID 来控制机器狗四条腿的动作，使每条腿的运动能够精确反映目标位置的变化。

（2）XYLineToAngel： 该函数解析输入的动作命令行 cmdline，提取每条腿的目标坐标（x, y），并通过逆解算法计算出对应的舵机角度 A1 和 A2。接着调用 SetAngelByID 来设置舵机角度，实现每个动作的执行。该函数还返回动作的延迟时间，确保每个动作按正确的时间顺序执行。

（3）DogAction： 该函数执行一个动作序列，遍历传入的动作列表（ActionList），依次解析每一条动作命令，并执行相应的舵机角度设置。每个动作会根据其延时执行，确保机器狗的每个动作间有正确的时间间隔。

（4）DogRun, DogBack, DogLeft, DogRight, DogDown, DogUp, DogSquat, DogBow： 这些函数分别实现了机器狗的前进、后退、左转、右转、趴下、站立、蹲下和鞠躬等基本动作。每个函数通过调用 DogAction 并传入相应的动作命令序列来控制机器狗的运动。

（5）DogWave： 该函数实现了机器狗的挥手动作。首先，机器狗执行一个蹲下动作，然后通过控制特定舵机角度，完成挥手动作。挥手动作会反复执行，直到接收到新的命令或外部中断。


2.2.3 PCA9685 驱动模块程序设计

模块说明：

本模块实现了对 `PCA9685 REF _Ref196061866 \r \h  \* MERGEFORMAT [4]` 舵机控制芯片的操作，包括设置输出频率、控制舵机角度、初始化和配置等功能。通过 I2C 通信协议与主机进行数据交互，实现对多个舵机的精确控制。

函数逻辑设计：

（1）PCA9685_Write： 该函数通过 I2C 总线向 PCA9685 设备写入指定的地址和数据。写操作的目标是向设备寄存器发送控制指令，以进行参数设置或状态修改。

（2）PCA9685_Read： 该函数通过 I2C 总线读取指定地址的寄存器数据。在通信过程中，首先发送读命令，然后接收来自 PCA9685 的数据。该函数常用于获取设备状态或寄存器的值。

（3）PCA9685_setPWM： 该函数用于设置指定舵机通道的 PWM 输出。通过计算得到 on 和 off 时间，控制舵机的开关周期。此函数是控制舵机角度的基础。

（4）PCA9685_setFreq： 该函数设置 PCA9685 的输出频率。频率的计算依据 PCA9685 内部振荡器的频率，通过预分频器值进行调整。该函数确保舵机能在设定的频率下工作，从而实现平稳控制。

（5）PCA9685_init： 该函数负责初始化 PCA9685，包括设置输出频率和初始舵机角度。通过调用 PCA9685_setFreq 和 PCA9685_setAllAngle 函数来完成初始化设置。

（6）PCA9685_setAngle： 该函数设置单个舵机的角度。舵机的角度由 0 到 180 度的范围内的输入值进行控制。通过计算，获取对应的 PWM 脉冲宽度并发送给 PCA9685，从而调整舵机的角度。

(7)PCA9685_setAllAngle：该函数设置所有舵机的角度与PCA9685_setAngle 类似，它会对所有 16 个舵机通道设置相同的角度值，用于统一的角度控制。

(8)AngleMove： 该函数根据目标角度和当前角度，逐步调整舵机的角度，确保舵机的运动平滑且可控。通过计算每个舵机当前角度与目标角度之间的差值，并根据步长（Step[]）决定每次移动的角度。该函数实现了多通道舵机的逐步控制，能够平滑地过渡到目标角度。


2.2.4 WS2812 驱动模块设计

模块说明：

本模块实现了对 WS2812 地址可控 LED 灯的控制。通过精确的时序控制信号，控制每个 LED 显示颜色，颜色数据采用 GRB 顺序。该模块通过 STM32F407VE 微控制器的 GPIO 端口，通过生成精确的时序信号，控制 WS2812 的显示效果。

函数逻辑设计：

（1）ws2812_WriteBit0： 该函数用于控制发送数据“0”位。通过精确控制高电平和低电平的持续时间（220ns ~ 380ns 和 580ns ~ 1.6μs），实现 WS2812 LED 的 bit 0 写入。

（2）ws2812_WriteBit1： 该函数用于控制发送数据“1”位。通过精确控制高电平和低电平的持续时间（580ns ~ 1.6μs 和 220ns ~ 420ns），实现 WS2812 LED 的 bit 1 写入。

（3）ws2812_Write24Bit： 该函数通过调用 ws2812_WriteBit1 和 ws2812_WriteBit0，依次发送 24 位的 RGB 数据（按照 GRB 顺序）。数据的最高位（位 23）首先发送，以此类推，最终发送最低位（位 0）。该函数用于发送 1 个 LED 灯的颜色数据。

（4）show_16GRB： 该函数用于更新 5 个 WS2812 LED 的显示效果。传入一个包含 5 个颜色值（GRB 顺序）的数组，通过循环调用 ws2812_Write24Bit 来逐个更新 LED 的显示颜色。该函数在每次更新时会保证数据写入过程的原子性，防止在更新过程中出现数据冲突。


2.2.5主程序框架设计

串口通信：USART1：用于 USB 转串口；USART2：蓝牙串口（用于遥控控制）；UART4：ESP32 模块（语音识别控制），定义了一些简单的控制命令（如前进、后退、左转、右转等）来控制机器人狗的运动。

舵机控制：使用PCA9685 PWM驱动板控制舵机的角度，通过I2C总线与STM32通信,根据接收到的命令调整舵机角度，控制机器人的四肢运动。通过PCA9685_setAngle函数设置舵机的角度，舵机的ID与角度值来自于解析的命令。当接收到控制命令（如前进、后退）时，程序根据命令确定需要调整的舵机角度。然后，程序通过 I2C 总线与 PCA9685 模块通信，设置相应舵机的角度，从而实现机器人的运动。

LED显示和流水灯：通过流水灯效果.LED灯的亮灭来展示机器人的状态。

接收和解析命令：蓝牙模块（USART2）接收来自遥控器的命令，使用连接到wifi后使用简单的网页进行控制，从而调整机器狗的前进和后退等操作。语音识别模块（UART4）接收来自 ESP32 的语音命令，使用相应的语音进行控制，如前进后退等进行编写的指令信息。

通过这个主程序框架，机器人的行为得到了有效的控制。串口通信模块确保外部设备可以传递控制信号，舵机控制模块通过 PCA9685 驱动板来实现机器人运动，LED 显示则用于直观地展示机器人的状态。

每个模块的协作使得机器人能够根据遥控或语音命令做出相应的反应，通过舵机的控制实现复杂的运动，而 LED 显示则为用户提供清晰的视觉反馈，展示机器人的工作状态。


3.总结


3.1所遇到的问题及解决办法

遇到的问题：

（1）串口不显示

解决办法：在询问老师后解决，解决办法是，给机器狗插电驱动识别需要时间，或者需要电池供电来唤醒驱动。

（2）舵机的调整

机器狗腿的运动需要正确的姿态，否则无法完成各种动作。

解决方法：通过调试参数获得正确的偏置（目标角度-基准角度）



（3）狗头的连接错误，狗头除VCC和GND，两个IO接口错误也会导致无法传递指令。

解决方法：按照对应的指导书上的接口完成连接。



（4）机器狗的WiFi连接后访问192.168.4.1失败

解决方法：最好的解决方式是关掉数据连接和网络切换，因为手机开启了这两个功能会让无法访问互联网的WiFi断开，数据加速也会导致连接网络时候使用数据流量访问192.168.4.1。
